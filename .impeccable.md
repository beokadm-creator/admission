# Design Context

## Essential (Post-Compact)

- 기본 인상은 공식적이고 안정적이어야 한다.
- 학교 공개 페이지는 모바일 우선으로 설계한다.
- 핵심 CTA와 현재 상태는 한 화면에서 즉시 이해되어야 한다.
- 과도한 장식보다 신뢰감, 명료성, 행동 유도를 우선한다.

<!-- STATIC:START -->
## Brand Direction

- 톤: 공식적, 안정적, 명확함
- 사용자 감정 목표: 불안 감소, 절차 신뢰, 다음 행동 명확화
- 기본 컬러 축: `#003B71` 중심의 차분한 블루 계열
- 보조 상태 컬러: 성공은 녹색, 경고는 앰버, 마감/오류는 로즈 계열

## UX Principles

- 대기 상태, 입장 가능 상태, 마감 상태를 즉시 구분할 수 있어야 한다.
- 본문 텍스트는 16px 이상을 기본으로 한다.
- 모바일 버튼 높이는 최소 56px을 유지한다.
- 학교 페이지는 낯선 사용자가 처음 방문해도 이해할 수 있는 언어를 사용한다.
- 공공기관형 신뢰감은 유지하되 지나치게 딱딱하지 않게 구성한다.

## Avoid

- 스타트업 랜딩 페이지 같은 가벼운 톤
- 과한 그래디언트, 네온, 과장된 입체감
- 중요 상태보다 장식이 먼저 보이는 레이아웃
- 모바일에서 작은 터치 타깃과 긴 가로 스크롤

## Design Context

### Users
**학부모 (40~50대)** + **학생 (10~20대)**. 주로 모바일로 접속하며, 입학 신청이라는 긴장된 상황에서 방문한다. 처음 방문해도 즉시 어떤 서비스인지, 지금 무엇을 해야 하는지 알 수 있어야 한다. 디지털에 익숙하지 않을 수 있는 학부모도 사용한다.

### Brand Personality
**신뢰·명료·격조** — 공공기관처럼 믿을 수 있되, 모던하고 사용하기 쉬운 느낌. 스타트업 랜딩이 아닌 공식 서비스의 인상. 사용자가 느끼는 불안을 줄이고, 절차에 대한 확신을 주는 것이 목표.

### Aesthetic Direction
- **레퍼런스**: Apple 공식 사이트, Linear — 여백 풍부, 타이포그래피 중심, 절제된 UI
- **폰트**: Pretendard (Google Fonts CDN) — 현대적 고딕, 높은 가독성
- **컬러**: `#003B71` 네이비 중심 단색 팔레트. 보조 강조는 최소화.
- **테마**: 라이트 모드 전용
- **안티 레퍼런스**: 스타트업 SaaS 랜딩, 과한 그래디언트, 네온, glassmorphism

### Design Principles
1. **위계가 먼저** — 접수 중인 학교가 히어로 수준으로 강조되어야 한다. 서비스 소개보다 현재 상태가 더 중요하다.
2. **여백으로 신뢰** — 빽빽한 정보보다 풍부한 여백이 공식 서비스의 무게감을 만든다.
3. **텍스트가 디자인이다** — 아이콘·카드 반복보다 잘 짜인 타이포그래피 위계가 더 효과적이다.
4. **모바일 우선** — 버튼 높이 56px+, 본문 16px+, 가로 스크롤 금지.
5. **상태를 즉시 이해** — 접수중(초록), 예정(앰버), 마감(로즈)을 색과 레이블로 명확히 구분.
