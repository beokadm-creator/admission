# 참가자 신청 접수 로직 설명서

> 본 문서는 시스템이 참가자의 신청을 어떤 순서로 처리하는지 단계별로 설명합니다.

---

## 한 줄 요약

**오픈 시간에 동시 접속 → 클릭 순서대로 번호 발급 → 설정된 동시 작성 인원만큼만 실시간으로 입장 관리 → 신청 완료 또는 시간 초과 시 다음 순번 자동 입장**

---

## 1. 오픈 대기

관리자가 설정한 **오픈 시간**이 되기 전까지 참가자는 대기 화면을 보게 됩니다. 오픈 시간이 되면 신청 버튼이 활성화됩니다.

> **참가자 화면**: "접수는 오픈 시에 시작됩니다" (오픈 전) → 신청 버튼 활성화 (오픈 후)

---

## 2. 대기번호 발급 (클릭 순서)

오픈 시간이 되면 모든 참가자가 동시에 신청 버튼을 누를 수 있습니다. 버튼을 누른 순서대로 서버가 **공식 대기번호**를 즉시 부여합니다.

- 번호 발급은 **1번, 2번, 3번...** 순서대로 자동 할당됩니다
- 이미 번호를 받은 참가자가 다시 눌러도 동일한 번호가 유지됩니다 (중복 발급 방지)
- 대기번호 발급에는 **상한**이 있습니다

> **상한 기준**: 정규 인원 + 예비 인원의 **1.5배**
>
> 예시) 정규 100명 + 예비 20명 = 120명 → 최대 180명까지 번호 발급
>
> 상한에 도달하면 새로운 번호 발급이 중단되고, 이후 접속자에게는 "마감 안내"가 표시됩니다.

> **참가자 화면**: "대기번호 N번을 받았습니다. 앞으로 M명 대기 중입니다."

---

## 3. 신청서 작성 입장 (동시 작성 인원 관리)

번호를 받은 참가자는 **순서대로** 신청서 작성 화면으로 입장합니다. 이때 관리자가 설정한 **동시 작성 가능 인원**을 기준으로 관리합니다.

> **예시**: 동시 작성 가능 인원이 **50명**이라면
> - 현재 신청서를 작성 중인 참가자가 **50명 미만**이면 → 다음 대기순번에게 즉시 입장 알림
> - 작성 중인 참가자가 **50명**이면 → 순서를 기다림

### 입장 조건

```
현재 작성 중인 인원 < 동시 작성 가능 인원 (예: 50명)
```

이 조건이 충족되면 **다음 대기순번 참가자가 즉시 입장**합니다. 누군가 신청을 완료하거나 시간 초과로 퇴장하면 빈자리가 생기는 즉시 다음 순번이 자동으로 올라옵니다.

> **참가자 화면**:
> - 대기 중: "현재 N명 대기 중입니다. 예상 대기시간 약 M분"
> - 입장 가능: "지금 바로 신청하기" 버튼 활성화

---

## 4. 신청서 작성 (3분 제한)

입장한 참가자에게 **3분**의 신청서 작성 시간이 주어집니다.

```
┌──────────────────────────────────────────┐
│           ⏱ 남은 시간: 02:45            │
│                                          │
│    이름: _______________                  │
│    연락처: _______________               │
│    이메일: _______________               │
│    주소: _______________                 │
│                                          │
│    [약관 동의]  [ 신청하기 ]             │
└──────────────────────────────────────────┘
```

### 시간 내 제출 완료 시

제출하면 서버가 현재 신청 인원에 따라 자동으로 상태를 결정합니다.

| 조건 | 결과 |
|------|------|
| 정규 인원에 **여석이 있으면** | **확정 (confirmed)** |
| 정규 인원이 찼고 예비 인원에 **여석이 있으면** | **예비 대기 (waitlisted)** — 순번 부여 |
| 정규 · 예비 모두 마감이면 | 접수 마감 안내 |

> **참가자 화면**:
> - 확정: "신청이 완료되었습니다!" (알림톡 발송)
> - 예비: "예비 신청 N번으로 접수되었습니다. 순서가 올라오면 알려드립니다." (알림톡 발송)

### 시간 초과 시 (3분 경과)

3분 내에 제출하지 못하면 해당 참가자의 **작성 자리가 즉시 반환**되고, 대기 중인 **다음 순번 참가자가 자동으로 입장**합니다.

> **참가자 화면**: "신청 시간이 만료되었습니다." → 메인 화면으로 이동

---

## 5. 대기 중 이탈 처리

신청서를 작성 중이거나 입장 대기 중인 참가자가 다음과 같은 경우 자동으로 대기열에서 정리됩니다.

| 상태 | 이탈 기준 |
|------|----------|
| **신청서 작성 중** | 3분 시간 초과 |
| **입장 대기 중 (waiting)** | 마지막 활동 후 **90초** 동안 반응 없음 |
| **입장 가능 상태 (eligible)** | 마지막 활동 후 **60초** 동안 반응 없음 |

이탈 처리된 참가자의 자리는 즉시 반환되며, **다음 순번 참가자가 자동으로 승격**됩니다.

---

## 전체 흐름도

```
참가자 접속
    │
    ▼
┌─────────────────┐
│  ① 오픈 대기     │ ← 오픈 시간까지 대기
└────────┬────────┘
         │ 오픈 시간 도달
         ▼
┌─────────────────┐
│  ② 대기번호 발급  │ ← 클릭 순서대로 1, 2, 3... 번호 부여
│  (상한: 정규+예비  │   상한 도달 시 마감
│   × 1.5배)       │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  ③ 순서 대기      │ ← 동시 작성 인원(예:50명)의
│  (waiting)       │   빈자리가 생길 때까지 대기
└────────┬────────┘
         │ 빈자리 발생 시 즉시
         ▼
┌─────────────────┐
│  ④ 신청서 작성    │ ← 3분 타이머 시작
│  (eligible →     │
│   reserved)      │
└────┬───────┬────┘
     │       │
     │ 3분 초과
     │       │
     │       ▼
     │  ┌──────────┐
     │  │ ⑤ 자리    │ → 다음 순번 자동 승격
     │  │    반환    │
     │  └──────────┘
     │
   제출 완료
     │
     ▼
┌─────────────────┐
│  ⑥ 결과 확정     │
│                  │
│  정규 여석 → 확정 │ → 알림톡 (확정)
│  정규 마감 → 예비 │ → 알림톡 (대기)
└─────────────────┘
```

## 예시 시나리오 (정규 100명, 예비 20명, 동시 작성 50명)

| 시간 | 상황 | 처리 |
|------|------|------|
| 00:00 | 오픈 시간 도달, 500명 동시 접속 | 버튼 활성화 |
| 00:00~00:05 | 클릭 순서대로 1~180번 발급 (상한 도달) | 180번 이후 접속자 마감 안내 |
| 00:05 | 1~50번 신청서 작성 시작 | 동시 작성 인원(50명) 가득 참 |
| 00:06 | 5번이 40초 만에 제출 완료 → 확정 | 빈자리 1개 발생 → 51번 자동 입장 |
| 00:07 | 12번이 3분 초과 → 작성 자리 반환 | 빈자리 1개 발생 → 52번 자동 입장 |
| 00:10 | 23번이 대기 중 90초 이탈 감지 | 대기열에서 제거 → 다음 순번 자동 승격 |
| 01:30 | 정규 100명 마감 | 이후 제출자는 예비 N번으로 접수 |
| 02:00 | 예비 20명 마감 | 접수 전체 종료 |

## 관리자 설정 항목 정리

| 설정 항목 | 설명 | 영향 범위 |
|-----------|------|-----------|
| **오픈 시간** | 신청 접수가 시작되는 시각 | 버튼 활성화 시점 |
| **정규 인원** | 확정 접수 인원 | confirmed 판정 기준 |
| **예비 인원** | 대기 접수 인원 | waitlisted 판정 기준 |
| **동시 작성 가능 인원** | 동시에 신청서를 작성할 수 있는 최대 인원 | 입장 속도 조절 (예: 50명 = 항상 50명이 작성 중인 상태 유지) |

---

## 안전 장치

| 장치 | 설명 |
|------|------|
| **번호 중복 발급 방지** | 같은 사용자가 여러 번 클릭해도 동일한 번호 유지 |
| **실시간 동기화** | 누군가 신청을 완료하거나 이탈하면 즉시 다음 순번이 자동 입장 |
| **이탈 자동 감지** | 대기 중 또는 작성 중 화면을 이탈한 참가자를 자동 정리 |
| **시간 초과 자동 반환** | 3분 내 미제출 시 자리가 즉시 다음 참가자에게 전달 |
| **정기 정리 (1분 주기)** | 서버에서 1분마다 만료된 세션과 이탈자를 자동 점검 |
| **알림톡 발송** | 확정, 예비 대기, 순번 승격 시 참가자에게 알림톡 자동 발송 |
